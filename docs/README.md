# 자동차 경주 게임

## 기능 목록
1. [x] 자동차 클래스를 생성한다.
   1. [x] 자동차는 이름과, 위치값을 가진다.
      1. [x] 자동차 이름 클래스를 생성한다.
         1. [x] 자동차 이름은 blank값이 될 수 없다.
         2. [x] 자동자 이름은 5글자 이하만 가능하다.
         3. [x] 입력받은 자동차 이름의 앞, 뒤 공백을 제거하고 인스턴스를 생성한다.
      2. [x] 자동자 위치 클래스를 생성한다.
         1. [x] 자동차 위치는 음수가 될 수 없다.
2. [x] 자동차 경주에 참여하는 자동차들을 리스트로 가지는 일급컬렉션을 생성한다.
   1. [x] 문자열을 ","을 구분자로 하여 입력받고, 자동차 인스턴스를 생성한다.
3. [x] 자동자 경주를 진행할 횟수를 나타내는 클래스를 생성한다.
   1. [x] 자동자 경주를 진행할 횟수는 숫자값이 입력된다.
   2. [x] 자동자 경주를 진행할 횟수는 양수만 가능하다.
4. [] 자동차 경주를 수행할 클래스를 생성한다.
   1. [] 경주 횟수에 따라 자동차를 움직인다.
      1. [] 자동차 경주를 위해, 자동차 위치 클래스에 메시지를 보내 자동차가 움직이도록 한다.
         1. [x] 랜덤 값을 통해 0~9 숫자 중 4이상일 경우 전진시킨다.
         2. [] 랜덤 값을 테스트하기 어렵기 때문에 테스트할 수 있는 구조로 만든다.
            1. [] 랜덤값을 생성하는 형태가 아닌, 랜덤값을 외부에서 주입 받도록 한다.
            2. [] 랜덤값을 외부에서 전략패턴을 통해 전달 받도록 한다.
         3. [] 자동차 위치 값을 매번 생성하지 않고, 캐싱을 사용하도록 변경한다.
   2. [] 자동차 경주를 진행하며 중간에 진행상황을 출력할 수 있도록 구조를 만든다.
      1. [] 경주 횟수를 하나씩 줄여가며 경주를 진행한다.
      2. [] 경주 횟수가 0이 되면 자동차 경주가 끝난 것으로 판단한다.
5. [] 자동차 경주의 우승자 클래스를 생성한다.
   1. [] 자동차 경주 게임의 결과 값을 받고, 우승자를 찾는다.
6. [] Application 클래스의 main() 메서드를 구현한다.
   1. [] 화면을 그리는 View 클래스를 생성한다.
   2. [] 로직을 처리하는 controller, service 클래스를 생성한다.
